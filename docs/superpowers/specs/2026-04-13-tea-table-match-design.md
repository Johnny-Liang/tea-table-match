# 茶馆组局系统设计

## 概述

帮助茶馆妈妈快速为客人安排座位，自动检测金额、抽烟、社交等约束冲突。

## 核心概念

| 概念 | 说明 |
|------|------|
| 客人 | 打牌的顾客，有金额偏好、抽烟偏好、社交排斥 |
| 桌子 | 4人位，用于落座 |
| 档位 | 5元、10元、20元三种茶费 |

## 数据模型

### 客人

```typescript
interface Guest {
  id: string;                    // 唯一标识
  name: string;                  // 姓名
  amounts: number[];             // 可接受档位 [5], [5,10], [10,20], [5,10,20] 等
  smokeTolerance: boolean;      // true=可接受抽烟, false=完全不接受
  excludeGuestIds: string[];    // 不能同桌的客人ID列表
  createdAt: number;             // 创建时间
  updatedAt: number;             // 更新时间
}
```

### 桌子

```typescript
interface Table {
  id: number;                    // 桌号 1-N
  guests: Guest[];               // 当前落座的客人（最多4人）
  amount: number | null;         // 当前桌统一档位（确定后不可改）
}
```

## 页面结构

### 1. 首页

- 显示所有桌子及其落座情况
- 每桌显示：桌号、落座客人列表（姓名+档位）、剩余位置
- 客人卡片可点击：点击后弹出操作菜单
  - 「重新派桌」：弹出派桌结果页面（检测所有桌子兼容性），选择新桌子后完成迁移
  - 「取消落座」：将该客人从桌子移除
- 底部固定按钮：「添加客人」和「下班」

### 2. 添加客人页面

**搜索客人**
- 输入框：模糊搜索客人姓名
- 搜索结果列表：点击选中
  - 已有客人 → 填充已有信息，可修改
  - 新客人 → 清空表单，待新建

**填写/修改信息**
- 姓名（必填）
- 茶费档位：5元、10元、20元（多选）
- 是否接受抽烟：☑ 接受 / ☑ 不接受（默认可接受）
- 排除客人：多选列表（所有已存客人）

**自动派桌按钮**
- 点击后根据约束搜索所有兼容桌子

### 3. 派桌结果页面

- 显示所有桌子列表（按兼容性排序：合适的在前）
- 每桌显示：桌号、已落座客人、已占位置/总位置、当前档位
- 合适的桌子：正常显示
- 不合适的桌子：显示为灰色/禁用状态，并标注原因（人数已满/金额不兼容/抽烟冲突/社交冲突）
- 用户可点击任意桌子灵活派桌

#### 智能迁移提示

当所有桌子都不合适时，系统自动检测是否存在可通过迁移1-2个客人来腾出合适位置：
- 遍历桌子组合，尝试移动1个或2个客人到其他桌
- 若移动后能腾出合适位置，显示提示：
  - "移动 [客人A] 从 桌1 到 桌3，可空出桌1"
  - 用户确认后自动执行迁移并派桌

### 4. 确认派桌

- 显示该桌详情和落座后效果
- 确认后：
  - 客人落座
  - 若桌子未确定档位，自动确定（取所有客人档位的交集）
  - 更新客人信息（新建或修改）

## 约束规则

派桌时必须同时满足以下条件：

1. **人数约束**：桌子未满（<4人）

2. **金额约束**：所有落座客人的档位必须有交集
   - 例如：客人A [5,10] + 客人B [10] = 交集 [10] ✓
   - 例如：客人A [5] + 客人B [10] = 无交集 ✗

3. **抽烟约束**：若新客人不接受抽烟，则同桌所有人必须都能接受抽烟
   - 即：有一人 `smokeTolerance=false` → 全桌人都必须是 `true`

4. **社交约束**：新客人与同桌所有人不能有互相排斥
   - 检查 `excludeGuestIds` 双向关系

## 交互流程

```
首页 → 点击"添加客人" → 搜索/输入客人 → 填写信息 → 点击"自动派桌"
     → 显示兼容桌子列表 → 点击选择桌子 → 确认派桌 → 返回首页
```

## 技术实现

- 单页Web应用
- 数据存储：localStorage（浏览器本地）
- 框架：原生HTML/CSS/JS 或 轻量框架
